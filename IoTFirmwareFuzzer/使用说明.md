# IoT Firmware Fuzzer 使用说明

## 一、环境准备

### 1. 安装 Python
- 需要 **Python 3.10 或更高版本**。
- 在命令行输入 `python --version` 或 `python3 --version` 检查。

### 2. 安装依赖
在项目根目录 `IoTFirmwareFuzzer` 下执行：

```bash
pip install -r requirements.txt
```

或使用国内镜像加速：

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 3. 可选工具（真实固件测试时才需要）
- **QEMU**：用于运行 ARM/MIPS 等架构程序。[下载](https://www.qemu.org/download/)
- **Binwalk**：用于解包固件。`pip install binwalk` 或从官网安装。
- **GDB**：用于崩溃时获取调用栈（可选）。

---

## 二、基本命令

所有命令都在项目根目录下执行，且以 `python main.py` 开头。

### 查看帮助与版本
```bash
python main.py --help
python main.py --version
```

### 查看当前配置
```bash
python main.py info
```
会显示：协议类型、目标架构、输出目录等。

### 生成初始种子
在正式 fuzz 前，建议先为对应协议生成一批合法报文作为“种子”：

```bash
# 为 MQTT 生成 20 个种子，保存到默认目录 ./seeds/mqtt
python main.py seed mqtt -n 20

# 为 HTTP 生成 10 个种子，并指定输出目录
python main.py seed http -n 10 -o ./my_seeds

# 为 CoAP 生成种子（当前实现会生成到 ./seeds/coap，需先有 coap 子目录或代码支持）
python main.py seed mqtt -n 15
```

生成后，种子在 `./seeds/<协议名>/` 下（如 `./seeds/mqtt/`）。

---

## 三、启动模糊测试（fuzz）

### 方式 1：不跑真实程序（推荐先这样试）
不启动 QEMU、不连真实设备，只用程序自带的“存根”执行器跑主循环，用于**验证框架是否正常、看报告是否生成**：

```bash
python main.py fuzz --no-qemu --protocol mqtt --timeout 60
```

- `--no-qemu`：不启动 QEMU，用内置存根执行。
- `--protocol mqtt`：按 MQTT 协议做变异。
- `--timeout 60`：跑 60 秒后自动停止（0 表示一直跑，需手动 Ctrl+C 停止）。

### 方式 2：指定固件文件（自动解包）
如果你有一个固件镜像（如 `.bin`、`.img`），程序会用 Binwalk 解包并尝试找目标程序：

```bash
python main.py fuzz -f D:\firmware.bin -a arm -p mqtt -t 300
```

- `-f`：固件文件路径。
- `-a arm`：目标架构（arm / mips / mipsel / aarch64 / x86 / x86_64）。
- `-p mqtt`：协议类型。
- `-t 300`：运行 300 秒。

### 方式 3：直接指定要测试的二进制
已经解包好、知道目标程序路径时：

```bash
python main.py fuzz -b /path/to/mosquitto -a x86_64 -p mqtt -t 600
```

- `-b`：目标可执行文件路径（例如 MQTT broker、HTTP 服务等）。
- 若目标为 ARM，需本机已安装 `qemu-arm`，并可用 `-a arm`。

### 常用参数汇总
| 参数 | 简写 | 说明 | 示例 |
|------|------|------|------|
| `--config` | `-c` | 配置文件路径 | `-c ./config/default_config.yaml` |
| `--firmware` | `-f` | 固件镜像路径 | `-f firmware.bin` |
| `--binary` | `-b` | 目标二进制路径 | `-b ./app` |
| `--protocol` | `-p` | 协议：mqtt / coap / http | `-p mqtt` |
| `--arch` | `-a` | 架构 | `-a arm` |
| `--timeout` | `-t` | 运行秒数，0=无限 | `-t 3600` |
| `--output` | `-o` | 结果输出目录 | `-o ./my_output` |
| `--no-rl` |  | 关闭强化学习调度 | 加上即生效 |
| `--no-qemu` |  | 不启动 QEMU（存根模式） | 加上即生效 |

## 四、运行中与结束

- **运行中**：终端会每隔约 5 秒打印一行统计，例如：  
  `执行 12345 速度 100/s 崩溃 0 语料 20 00:02:05`
- **停止方式**：  
  - 若设置了 `-t 60`，到时间会自动停止。  
  - 否则在终端按 **Ctrl+C** 可优雅退出（会先写报告再退出）。
- **结果位置**（默认在 `./output/`）：
  - `output/crashes/`：崩溃输入（`.bin`）及元信息（`.json`）。
  - `output/reports/`：HTML/JSON 等报告（如 `report_20260227_195600.html`）。
  - `output/logs/`：运行日志。

用浏览器打开 `output/reports/` 下最新的 `.html` 文件即可查看测试报告。

---

## 五、修改默认配置

编辑 **config/default_config.yaml** 可长期修改默认行为，例如：

- `fuzzer.output_dir`：结果输出目录。
- `protocol.name`：默认协议（mqtt / coap / http）。
- `emulation.architecture`：默认架构。
- `emulation.timeout`：单次执行超时（秒）。
- `seed.init_corpus`：种子目录（需与 `python main.py seed` 的输出一致）。
- `expert.rl_enabled`：是否启用强化学习调度。
- `report.format`：报告格式（html / json）。

命令行里指定的参数会覆盖配置文件中的对应项。

## 六、推荐使用流程

1. **第一次使用（验证环境）**
   ```bash
   pip install -r requirements.txt
   python main.py seed mqtt -n 10
   python main.py fuzz --no-qemu --protocol mqtt --timeout 30
   ```
   然后打开 `output/reports/` 下生成的 HTML 报告，确认有统计与崩溃列表（可能为 0）。

2. **有固件时**
   ```bash
   python main.py fuzz -f 你的固件.bin -a arm -p mqtt -t 3600 -o ./output_arm
   ```

3. **有现成二进制时（如本机 x86 程序）**
   ```bash
   python main.py fuzz -b ./你的程序 -a x86_64 -p mqtt -t 1800
   ```

按上述步骤即可完成从安装、生成种子到跑 fuzz、看报告的全流程。若某一步报错，把完整报错信息贴出来可继续排查。

## 七、Web 界面（推荐给不熟悉命令行的用户）

本程序提供浏览器版控制界面，可在页面上选择协议、超时、是否存根模式等，一键开始/停止测试并查看实时统计与日志。

### 启动方式
```bash
pip install flask
python main.py ui
```
或直接运行：
```bash
python web_ui.py
```

启动后终端会提示访问地址（默认 **http://127.0.0.1:5000**）。用浏览器打开该地址即可。

### 界面功能
- **测试配置**：协议（MQTT/CoAP/HTTP）、架构、超时时间、目标二进制/固件路径、输出目录；勾选「存根模式」可不连 QEMU 先体验。
- **开始 / 停止测试**：点击「开始测试」后，下方会显示实时执行次数、速度、崩溃数、语料库大小、运行时长；点击「停止测试」可提前结束。
- **运行日志**：页面下方会滚动显示最近日志，便于排查问题。
- **打开最新报告**：测试结束后会出现「打开最新报告」链接，点击可在新标签页查看 HTML 报告。
- **生成种子**：在「生成种子」区域选择协议和数量，点击「生成种子」即可在默认 `./seeds` 下生成初始语料，无需再敲命令行。

其他电脑访问同一局域网时，可用 `http://<本机IP>:5000` 打开（需本机防火墙放行 5000 端口）。
